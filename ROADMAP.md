## 当前状态

v0.1.0 核心系统基础已发布。离线战斗、幸存者玩法、野外地图、剧情演出、实时联机等方向正并行推进，部分方向已有可玩雏形。

---

## 方向与进度

### 1. 架构职责治理

- **目标**：减少全局状态桶与隐式流程依赖，让离线副本、实时副本、幸存者模式都能基于清晰的数据所有权继续扩展。
- **路线**：拆清 `GameStateModel` 的全局阶段、副本与结算状态 → 进入副本、撤离、结算、回大厅等流程收口到明确 Command/System → 场景构建使用显式上下文。
- **进度**：部分完成。副本与结算状态已有拆分方向，仍需迁移引用、收口流程，并拆分离线与实时副本场景构建上下文。

### 2. 离线战斗与离线副本

- **目标**：为普通离线副本、幸存者模式及后续战斗玩法提供稳定的底层战斗能力。
- **路线**：补齐离线战斗基础闭环与实体子弹链路 → 表现同步增量化、通用战斗性能基础、核心炮塔通用能力 → 统一离线副本胜负与结算。
- **进度**：部分完成。离线战斗入口与基础规则已存在；实体子弹同步、刷新、移除与同帧命中保护，表现同步增量化与通用性能基础待推进。

### 3. 幸存者玩法

- **目标**：让幸存者模式从基础可玩推进到有节奏、有构筑、有复玩目标的独立玩法。
- **路线**：稳定模式基础与性能容量 → 调整目标时间、经验曲线、专用波次与 Boss 节奏 → 扩展武器构筑、局外记录与长期成长。
- **进度**：部分完成。基础模式、经验与升级选择已有雏形；目标时间配置化、前期升级反馈、专用波次、Boss 节奏、结算摘要与局外成长待补齐。

### 4. 地图编辑器与野外地图

- **目标**：让地图从可编辑的格子数据升级为能支撑探索、资源、敌人、风险收益与撤离路线判断的野外地图。
- **路线**：地图数据、运行时基础、野外地貌生成与编辑器区域化生产已完成 → 区域玩法分布逻辑与玩法健康度（已完成）→ 表现增强、正式美术与性能优化。
- **进度**：前三阶段已完成。区域玩法分布逻辑已落地：地貌玩法语义、资源 / 敌人 / 危险点稳定生成（生成结果为不可变快照）、风险收益评分与健康度判断、编辑器玩法预览，均有 EditMode 测试覆盖。阶段4 已接入生产校验：模型层 `MapGameplayProductionValidator` 作为唯一生产校验入口，编辑器在「生成」前后展示玩法健康度（不阻塞导出）；批量校验与预览分项检查暂延后。玩法点仍为逻辑层，未写入文档。下一步为表现增强、正式美术与性能优化，以及与幸存者 / 掉落 / 怪物配置的数值对接。

### 5. 剧情演出系统

- **目标**：把剧情从静态 NPC 对话扩展为可配置、可验证、可逐步增强的演出流程。
- **路线**：补齐剧情数据源与轻量 `StoryPerform` 播放器 → 非阻塞播放、Dialogue 步骤、角色转向/移动与基础镜头表现 → 任务/区域触发、特效音效与复杂过场。
- **进度**：部分完成。对话系统与基础播放器已接入；Luban 源链路、非阻塞播放、Dialogue 步骤与真实角色表现待完成。

### 6. 实时联机

- **目标**：提升联机代码可维护性，解决实时移动卡顿与远端玩家抖动。
- **路线**：整理 Net/Handler 业务入口 → 诊断移动同步质量 → 为远端玩家加入 snapshot buffer 插值 → 必要时本地预测与服务端校正 → 继续拆分协议处理结构。
- **进度**：部分完成。基础联机结构与命名方向已有；移动同步诊断、远端插值与本地预测待完成。

## 近期优先级

1. **离线战斗实体子弹闭环** —— 幸存者与普通离线战斗表现的共同基础。
2. **幸存者目标时间、经验曲线与专用波次** —— 最快改善当前可玩性。
3. **地图玩法分布的表现层接入** —— 区域玩法分布逻辑（资源 / 敌人 / 危险点 + 风险收益健康度）已完成，下一步把逻辑点接入正式资源、怪物与掉落配置。
4. **GameStateModel 职责收口** —— 减少后续玩法扩展时的全局状态耦合。
5. **实时联机移动同步诊断与远端插值** —— 先量化问题，再做表现策略。
